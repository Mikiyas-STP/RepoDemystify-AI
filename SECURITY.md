# Security Policy

## Overview

Security is important for RepoDemystify AI because the project may process software repositories and developer information.

This document explains how security issues should be reported.

---

# Supported Versions

Currently supported:

| Version | Supported |
|---|---|
| Latest development version | Yes |

Older versions may not receive security updates.

---

# Reporting a Security Issue

If you discover a security vulnerability:

Please do not create a public GitHub issue.

Instead, report it privately through the repository security reporting feature.

Include:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested mitigation if available

---

# Examples of Security Issues

Examples include:

- Exposed credentials
- Authentication vulnerabilities
- Data leakage
- Unsafe repository processing
- Dependency vulnerabilities

---

# Responsible Disclosure

Please allow maintainers reasonable time to investigate and fix security issues before public disclosure.

---

# Security Considerations

RepoDemystify AI will consider:

- protection of user data
- secure handling of repositories
- API security
- dependency management
- secret management

---

Thank you for helping keep the project secure.
