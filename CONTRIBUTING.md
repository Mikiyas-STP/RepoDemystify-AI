# Contributing to RepoDemystify AI

Thank you for your interest in contributing to RepoDemystify AI.

RepoDemystify AI is an open-source project focused on helping developers understand unfamiliar software repositories through AI-powered analysis.

We welcome contributions from developers of all experience levels.

---

# How You Can Contribute

There are many ways to contribute:

## Code Contributions

Examples:

- Backend development
- Frontend improvements
- AI integration
- Repository analysis features
- Testing improvements

---

## Documentation Contributions

Examples:

- Improving explanations
- Adding tutorials
- Fixing documentation errors
- Creating examples

---

## Research Contributions

Examples:

- Evaluating AI responses
- Improving prompts
- Exploring repository analysis techniques

---

# Getting Started

Before contributing:

1. Read the project documentation.

Start with:

```
docs/PROJECT_VISION.md
docs/ARCHITECTURE.md
docs/ROADMAP.md
```

2. Set up the development environment.

See:

```
docs/DEVELOPMENT_GUIDE.md
```

3. Choose an issue or create a discussion.

---

# Development Workflow

## 1. Fork the Repository

Create your own copy of the project.

---

## 2. Create a Branch

Use descriptive branch names.

Examples:

```
feature/repository-scanner

feature/ai-chat

fix/database-connection

docs/improve-installation
```

---

## 3. Make Your Changes

Keep changes:

- focused
- readable
- well documented

Avoid combining unrelated features in one pull request.

---

## 4. Test Your Changes

Before submitting:

Backend:

```
./mvnw test
```

Frontend:

```
npm test
```

---

## 5. Commit Changes

Use meaningful commit messages.

Format:

```
type: description
```

Examples:

```
feat: add repository analysis service

fix: resolve database connection issue

docs: update contribution guide

test: add repository service tests
```

---

# Pull Request Guidelines

A good pull request should include:

## Description

Explain:

- what changed
- why it changed
- how it improves the project

---

## Testing

Explain:

- what tests were added
- how the changes were verified

---

## Screenshots

For UI changes, include screenshots where appropriate.

---

# Code Style

## Java

Follow:

- clean code principles
- meaningful naming
- single responsibility principle
- object-oriented design principles

---

## React

Follow:

- reusable components
- clear component responsibilities
- maintainable state management

---

# Issues

Before creating an issue:

- search existing issues
- provide clear details
- include reproduction steps when relevant

---

# Questions

If you are unsure:

Open a discussion.

Good questions help improve the project for everyone.

Contributors should:

- respect others
- communicate clearly
- give constructive feedback
- help others learn
